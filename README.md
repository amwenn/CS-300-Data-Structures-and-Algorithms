# CS 300 Data Structures and Algorithms Portfolio

## Project Overview
The main problem I was solving in this course was creating programs that could efficiently organize, store, search, and display large amounts of data using different data structures and algorithms. Throughout the course, I compared vectors, hash tables, and binary search trees to determine which data structure best fit a given problem. In Project Two, I developed a course advising program that reads course information from a file, stores the data in a binary search tree, prints an alphanumeric list of courses, and allows users to search for a specific course and view its prerequisites.

## Approach
I approached each project by breaking the problem into smaller pieces before writing any code. First, I analyzed the requirements and designed pseudocode to understand the program flow. Then I selected the data structure that best met the project requirements. Understanding data structures was important because each one has different strengths and weaknesses. After comparing vectors, hash tables, and binary search trees, I chose a binary search tree because it provides efficient searching while also allowing courses to be displayed in sorted order through an in-order traversal.

## Roadblocks
One of the biggest challenges I encountered was configuring my development environment and making sure the program could correctly locate and read the input files. There were also times when Visual Studio updates caused issues that required cleaning up and reinstalling components before I could continue working. I overcame these problems by troubleshooting one step at a time, verifying file locations, testing individual functions, and using the debugger to identify where errors were occurring. Breaking larger problems into smaller tasks helped me solve issues more efficiently.

## Software Design
Working on these projects helped me better understand how choosing the right data structure can greatly affect a program's performance and overall design. Before this course, I focused mainly on writing code that worked. Now I spend more time planning the program, selecting the most appropriate data structure, and considering the efficiency of different algorithms before I begin coding. This has helped me write programs that are more organized and easier to understand.

## Writing Maintainable Code
This course also improved the way I write code by encouraging better organization, meaningful variable names, modular functions, and clear comments. Separating tasks into individual functions made the programs easier to test, debug, and reuse. I also learned the importance of validating user input, handling errors, and writing readable code that another programmer can easily understand and maintain. These are practices I plan to continue using in future courses and professional projects.
